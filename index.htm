Https status code

Cartoons

Code Response Image Definition
100 Continue The HTTP 100 Continue informational status response code indicates that everything so far is OK and that the client should continue with the request or ignore it if it is already finished. To have a server check the request's headers, a client must send Expect: 100-continue as a header in its initial request and receive a 100 Continue status code in response before sending the body.
200 Success Standard response for successful HTTP requests. The actual response will depend on the request method used. In a GET request, the response will contain an entity corresponding to the requested resource. In a POST request, the response will contain an entity describing or containing the result of the action.
201 Created The HTTP 201 Created success status response code indicates that the request has succeeded and has led to the creation of a resource.
202 Accepted The request has been accepted for processing, but the processing has not been completed. The request might or might not be eventually acted upon, and may be disallowed when processing occurs.
203 Non Authorized The HTTP 203 Non-Authoritative Information response status indicates that the request was successful but the enclosed payload has been modified by a transforming proxy from that of the origin server's 200 (OK) response.
204 No content Although a request has been successful, the client is not required to leave the current page, according to the HTTP 204 No Content success status response code.
206 partial content When a client sends a Range header to request only a portion of a resource, this response code is utilized.
207 Multi status conveys details on a variety of resources for circumstances where a variety of status codes may be suitable.
208 Already reported a response component to prevent listing the internal elements of several bindings to the same collection repeatedly.
209 I'm used The answer shows what happened after one or more instance-manipulations were performed on the current instance after the server satisfied a GET request for the resource.
300 Multiple choice There are multiple viable answers to the request. One of them should be selected by the user agent or user. (There isn't a set mechanism to select one of the answers; HTML links to the options are suggested instead so the user can decide.)
301 Moved permanently The URL of the requested resource has been changed permanently. The new URL is given in the response.
302 Found This response code indicates that a temporary change has been made to the URI of the requested resource. The URI may undergo additional modifications in the future. Therefore, the client should continue to utilize this URI for all upcoming requests.
304 Not modified This is used for caching purposes. It tells the client that the response has not been modified, so the client can continue to use the same cached version of the response.
306 Unused This response code is now only reserved; it is no longer in use. It was made use of in the HTTP/1.1 specification's earlier iteration.
400 Bad request Due to something that appears to be a client fault (such as incorrect request message framing, incorrect request syntax, or deceptive request routing), the server cannot or will not perform the request.
401 Unauthorized Although "unauthorized" is specified in the HTTP standard, "unauthenticated" really refers to this response in semantic terms. In order to receive the specified response, the client must first verify themselves.
402 Payment required For usage in the future, this response code is set aside. Although using this status code for digital payment systems was the original intent, it is now incredibly little utilized and has no established practice.
403 Forbidden The server is refusing to provide the requested resource because the client does not have authorization to access the content. The client's identity is known to the server, unlike 401 Unauthorized.
404 Not found The requested resource is not on the server. This indicates that the URL is not recognized by the browser. This could also imply that a resource's endpoint is legitimate in an API even while the resource itself doesn't exist. In order to conceal the existence of a resource from an unauthorized client, servers may also deliver this response in place of the 403 Forbidden response.
406 Not allowed When the web server performs server-driven content negotiation but is unable to locate any material that meets the user agent's requirements, it sends this answer.
408 Request timeout Some servers may provide this response while the connection is idle, even in the absence of a client request. It indicates that the server wants to terminate this inactive connection. Since some browsers, like Chrome, Firefox 27+, or IE9, use HTTP pre-connection methods to speed up browsing, this answer is used much more frequently. Note that not all servers provide this message; instead, some just terminate connections.
410 Gone When the requested content has been permanently removed from the server, this response is provided without a forwarding address. It is expected of clients to take down any caches and connections to the site. The HTTP protocol states that "limited-time, promotional services" should use this status code. APIs shouldn't feel forced to use this status code to denote removed resources.
414 Url toolong The server is unwilling to interpret the entire URI that the client requested.
415 Unsupport media type The server is refusing the request because it does not support the requested data's media format.
418 I'am teapot The effort to make coffee in a teapot is rejected by the server.
423 Locked The resource that is being accessed is locked.
425 Too early Indicates that the server is unwilling to risk processing a request that might be replayed.
426 Upgrade required The server won't fulfill the request using the current protocol, but after the client switches to a new protocol, it could be willing to. The required protocol(s) are specified by the server in an Upgrade header in a 426 response.
429 Too many request Rate limiting" refers to the user sending too many requests in a specific length of time.
500 Internal server error The server has come upon a circumstance that it is unsure how to handle.
501 Not implemented The server does not support the request method, hence it cannot be processed. Servers are only required to provide GET and HEAD, which means they cannot return this code for any other methods.
503 Service unavailable The request cannot be handled by the server at this time. A server that is overloaded or down for maintenance are frequent causes. Be aware that a user-friendly page outlining the issue should be sent along with this reply. If possible, the Retry-After HTTP header should include the anticipated time before the service recovery, and this response should only be used in brief circumstances.
504 Gateway timeout When the server is serving as a gateway and cannot get a response in time, an error response is returned.
507 Insuffiecient storage The server is unable to store the representation required to correctly finish the request, hence the method could not be applied to the resource.
508 Loop detected While handling the request, the server discovered an infinite loop.
509 Not extended The server needs to extend the request in order to fulfill it.
